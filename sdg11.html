HOUSING AND REAL ESTATE SECTOR

Using Sorting Algorithms for Affordable Housing Analysis


Context:

In a growing urban environment, real estate prices fluctuate widely across neighborhoods. Citizens and urban planners need an efficient way to identify affordable housing options and high-value areas to make suitable decisions.

Problem Statement:

Given the list of properties within the city along with prices, area and house number. We need to sort all the properties in ascending order in order to find affordable property.

Possible solution:

Here we can use Sorting algorithms in order to sort all the buildings or houses according to price.
There are plenty of sorting algorithms that can be used in order to find the houses, But here i used Merge sort as
1)as the number of buildings, apartments, houses are very huge therefore merge can be used as it is provides stability on huge data.
2)as time complexity of merge sort is n(log n) which makes it highly efficient for this task.

Example


Scenario:
A real estate agent has the following house prices (in lakhs of ₹):
₹25,00,000, ₹10,00,000, ₹30,00,000, ₹15,00,000, ₹40,00,000
The agent wants to:
1. Sort the prices in descending order to identify premium houses.
2. Sort the prices in ascending order to identify affordable houses.
Explanation
Steps of Merge Sort:
1. Divide: Split the array into two halves recursively until each subarray has one element (which is inherently sorted).
2. Conquer: Sort the smaller subarrays by merging them.
3. Combine (Merge): Merge the sorted subarrays to form a single sorted array.
Output:
House prices in descending order (₹):
₹40 lakh
₹30 lakh
₹25 lakh
₹15 lakh
₹10 lakh
House prices in ascending order (₹):
₹10 lakh
₹15 lakh
₹25 lakh
₹30 lakh
₹40 lakh
